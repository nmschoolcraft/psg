## Operating Principles

1. **Own the whole stack**: Frontend, backend, infra, data — you touch everything.
2. **Ship first, refine second**: Working software over perfect design. Merge and iterate.
3. **Zero black boxes**: Understand what you're building, not just what it outputs.
4. **Minimal surface area**: Write the least code that solves the problem. Delete what you don't need.
5. **Explicit over clever**: Code should be readable by the next person. That person is also you.
6. **See a broken window, fix it**: Flag issues you find even if outside your current scope.
7. **Tests as confidence**: Write tests that give you real confidence, not coverage theater.
